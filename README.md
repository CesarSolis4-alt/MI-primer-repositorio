# Proyecto: Calculadora de Presupuesto Mensual

##  Descripción del caso
La Calculadora de Presupuesto Mensual es una aplicación diseñada para que un usuario pueda ingresar sus ingresos y gastos y obtener automáticamente su balance mensual.  
Este proyecto se desarrolló como una práctica de documentación en *Markdown, uso de **control de versiones con GitHub* y aplicación de *mantenimiento adaptativo* para asegurar que el sistema permanezca funcional ante cambios tecnológicos.

---

##  Objetivos del proyecto
- Permitir el registro de ingresos y gastos.  
- Calcular el balance mensual de manera automática.  
- Aplicar buenas prácticas de documentación en Markdown.  
- Implementar control de versiones mediante Git y GitHub.  
- Proponer un mantenimiento adecuado para garantizar la continuidad del software.

---

## Requerimientos del sistema

### *Requerimientos funcionales*
1. El sistema debe permitir ingresar los ingresos del usuario.  
2. Debe ser posible registrar gastos individuales.  
3. El sistema debe mostrar el balance total del mes.  
4. La interfaz debe ser clara y fácil de usar.

### *Requerimientos no funcionales*
1. Debe ejecutarse correctamente en navegadores actuales.  
2. La documentación del proyecto debe estar en formato Markdown.  
3. La calculadora debe responder sin errores en los cálculos.  
4. La interfaz debe adaptarse a distintos tamaños de pantalla.

## Tipo de mantenimiento propuesto  
El tipo de mantenimiento seleccionado para este proyecto es el *Mantenimiento Adaptativo*, ya que el sistema podría requerir ajustes debido a:

- Nuevas versiones de navegadores web.  
- Cambios en resoluciones o tamaños de pantalla.  
- Actualizaciones del sistema operativo.  
- Cambios en estándares HTML, CSS o JavaScript utilizados.

### Acciones de mantenimiento adaptativo recomendadas:
- Actualizar estilos para compatibilidad con modo oscuro.  
- Mejorar el diseño responsivo para móviles.  
- Ajustar el código a futuras versiones de navegadores.  
- Reemplazar funciones obsoletas por alternativas modernas.

##  Reflexión sobre el control de versiones

El uso de *Git y GitHub* permitió mantener un historial claro y profesional del proyecto, facilitando:

- Registrar cada avance mediante commits.  
- Restaurar versiones anteriores si es necesario.  
- Mantener la documentación organizada y accesible.  
- Comprender cómo evoluciona el proyecto con el tiempo.  
- Trabajar con mayor disciplina y orden, tal como se hace en proyectos reales.

El control de versiones es fundamental para el desarrollo moderno, ya que evita pérdidas de información y mejora la colaboración.

El proyecto de la Calculadora de Presupuesto Mensual demostró la importancia del mantenimiento adaptativo, la documentación clara en Markdown y el uso del control de versiones.
