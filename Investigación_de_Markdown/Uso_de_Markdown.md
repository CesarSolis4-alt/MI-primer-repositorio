# Uso de Markdown en proyectos de software

## ¿Qué es Markdown y por qué se utiliza?

*Markdown* es un lenguaje de marcado ligero creado por John Gruber y Aaron Swartz para redactar documentos estructurados de forma sencilla. Su sintaxis permite convertir el texto en formatos como *HTML, PDF o LaTeX*, lo que lo vuelve ideal para proyectos de software.

Según *Antoine Béchoux, en su artículo *“Boost your documentation with Markdown”, Markdown se ha convertido en una herramienta esencial dentro de la documentación técnica porque:

- Permite escribir documentación clara sin usar herramientas pesadas.  
- Facilita integrar fragmentos de código con resaltado.  
- Genera documentación profesional manteniendo la legibilidad del texto.  
- Se integra fácilmente con repositorios como GitHub, GitLab o Bitbucket.  

También destaca el uso de herramientas como *MkDocs* o *Docusaurus* que transforman archivos Markdown en sitios web completos para documentación.

## Ventajas de usar Markdown junto a GitHub

1. *Visualización inmediata:* GitHub renderiza automáticamente los archivos .md.  
2. *Control de versiones:* Cada cambio queda registrado mediante commits.  
3. *Colaboración eficiente:* Permite que varios desarrolladores editen la misma documentación.  
4. *Portabilidad total:* Funciona en cualquier sistema operativo.  
5. *Claridad profesional:* Mantiene una estructura limpia y estándar.
